Elements
A tag and content, like this: <p>This is an element</p>. Elements can either be block-level, or inline. A block-level element, always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can). Some examples include <p>,<H1-6><div>,<ul>.

An inline element does not start on a new line and only takes up as much width as necessary. Some examples include <a> and <img>.
Attributes
Attributes provide additional information about the contents of an element. They are made up of a name and a value. For example, the anchor tag (<a>) has an href attribute, where “href” is the name, and the link is the value. This website explains nicely.
Lists
There are three types of lists:
  1. Ordered lists, or <ol>, like this one.
  2. Unordered lists, or <ul>, which will create bullet points. Both of these use <li> tag to display the list items.
  3. Definition Lists. These use <dl> as the parent tag, and then <dt> for the definition term and <dd> for the definition definition (this entire site was created as a definition list).

Headers
Although headers decrease in size, they should be used to denote importance with <H1> being the most important, and <H6> being the least.
Adding Emphasis
There are two ways to bold or italicize: (1) If you are writing stylistically, use <b> or <i> (2) If you are trying to convey importance, or writing logically, use <em> or <strong>.
The HTML Sandwich
At the top will always be your doctype declaration (ex:<!doctype html>), but then after that comes the . . .
  1. Bread: <html lang="en"> (entire document) </html>
  2. Condements: <head>(invisible elements such as title, stylesheets, and meta tags)</head>
  3. Meat: <body>(visible content such as headers, tables, paragraphs, lists, or Van Gogh Paintings)</body>

Images

Adding images is easy peezy, but make sure to add the boring stuff, like this: <img src=".. /funny_pics/banana_peel" height="200" width="200" alt="Guy slipping on banana peel." title=“The oldest prank in the book”>You can also add height and width in CSS. This is for when the page loads before the picture. That way, the text will be structured correctly. The image should also fit the color palette of the page.

You can also get fancy and float the image to the left or the right. In this example, I've floated it right. In the img element, I added “class=floatRight,” and then in my CSS I wrote .floatRight {float:right; margin:20px;}

Also, this picture used to have a white border, but it looked way cooler without it. To take away the border, you have to cut it out in Photoshop and then save it as a .png. Saving it as a jpeg will not do the trick (by the way, “.png” is an extension. This is the suffix that goes after a file name to denote what type of file it is. The extension for an html file is either “.html” or “.htm.”).

Images are considered replaced content because they replace the element with the asset that is being referenced. Also, go ahead and hover over this. That's done using the title attribute, and what you are seeing is called the tooltip. This gives additional info about the picture. You also always want to include the alt attribute in case the picture doesn't load. If there's no meaning to the picture, and it's just decoration, still use alt, but just leave the quotes empty.

It's not a bad idea to create an images folder. If you have lots of photos, make subfolders of the categories.

Use a JPEG for any picture that has lots of colors. Use GIF for things like logos, where there are few colors or large areas of the same color. NOT for snow, sky, or grass. Save photos in the same dimensions as you'll use them. Portraits are taller than wide, and landscapes are wider than tall. ALWAYS work in pixels rather than inches. Animated GIFS are usually only used with simple illustrations because each photo increases the size of the file, which = longer load times.

Vector images are different from bitmap images because they're resolution independent. They're created by creating points, then drawing lines between them and filling in the colors. They're still usually saved as bitmap images on the web.

If you ever need to replace an image, right click it and choose “Open Image in New Tab.” The size will appear on the actual tab.

Tables
A table is essentially a grid. You enter in one cell of one row at a time. <tr> means table row, <td> means table data (each cell), and <th> means table heading. You can use the scope attribute within the table header element, and make sure to fill in even empty cells. For example:
Monday Tuesday
Hours Worked 8 12
Dollars earned $160 $240

If you need to span across multiple columns (like if you're drawing a schedule), use the attribute, colspan=“#of columns.” If you need to span multiple rows, use rowspan=“#of rows.” If you need to make a long table, wrap the entire header in <thead>, the body in <tbody>, and the footer in <tfoot>.

Forms
The <form> element typically has three attributes: (1) action (2) method (3) id.

The value of the action attribute is a URL to the site on the server that'll be receiving the collected data. The value of the method attribute is either ”get“ or ”post.“ Use the first for short forms, and the second for longer forms that may have sensitive data.

Adding Video

There are two ways: (1) Embedding the video from another site, (2) hosting the video on your own site. You can be charged extra for uploading lots of data to your site, but YouTube may run ads.

If you host it on your own site, you can use the <video> tag. Some common attributes are . . .

  • src—to give the path to the video, just like you would with a picture.
  • poster—the picture you want displayed while your videos is not playing.
  • width and height—you can find the dimensions by clicking “get info” on the video file.
  • preload—tells the browser when to download the video (none vs auto vs metadata)
  • loop—duh.
  • controls—tells the browser to supply its own controls for playback. If you don't include this, users may not be able to play the video.
  • autoplay—Yep.

To embed from YouTube, right click on the video and choose, ”copy embed code.“ You can then customize the width and height.

Adding Audio

Just like with videos, you can either embed or host your own audio. The attributes are the same, except for poster. Make sure to add some text in case the browser does not support the HTML5 audio element.

To embed from another site, just copy the embed code and then tweak as necessary.

Semantics
Semantics structuring, or page structure, is by far the most boring part of html. Here's the dilly . . .
  • <header role=“banner”> goes in front of any introductory information. Only use the role once, but use the header tag as much as you need.
  • <nav role=“navigation”> goes in front of the navigation section (links to other parts of the site)
  • <main role=“main”> goes in front of the meat and potatoes, or the main content of the site. In this site, it's all main because it's all important
  • <article role=“article”> is for any section that can stand on its own. Can you post it by itself as a website? Then use article. In this site, I used it for everything because it's all within the context of learning HTML.
  • <section> is for any section that you would put in an outline, but that cannot stand on its own.
  • <div> is for any section that has no semantic meaning. It is considered an anonymous tag and allows you to group a set of elements together. It is a block level element, so when you use it, the content will appear on the next line. People often add comments after the closing <div> tag. For example, </div><! – – end of comment section – –>
    You can add semantic meaning to div tags by adding class or Id articles. Ids can only be used once and should start with either an underscore or a letter. Classes can be used multiple times and the name usually hints at what its use is (ex:class=“important”). You can assign more than one class to the same element by separating them with spaces.
  • <span> is just like <div>, but is inline. It usually contains an id or class attribute so it can be used with CSS.
  • <aside=“complementary”> is for related content that is somewhat separate from the main content of the page. For example: sidebars, quotes, or ads.
  • <footer role=“content info”>is for the end section, where you may include ©TonyDaltorio, or your contact info (although he cautioned against adding your contact e-mail like this).
